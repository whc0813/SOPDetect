# 产品需求文档 (PRD)：基于视觉分析的标准操作流程管理系统

## 1. 项目概述
- **项目名称**：基于视觉分析的标准操作流程 (SOP) 管理系统
- **项目类型**：毕业设计
- **项目愿景**：开发一个基于视频分割和大视觉/语言模型（大模型）的标准行为识别系统。通过智能化的视觉分析技术，实现对人员操作行为的高效、准确识别与正误判断，从而推动相关领域（如工业制造、医疗操作、实验室规范等）的标准操作流程规范化与自动化管理。

## 2. 目标用户与角色
系统主要面向两类用户角色，具有不同的权限和业务流：
- **管理员 (Admin)**：负责制定和维护标准操作流程（SOP），定义每一步骤的具体规范和评判标准。
- **普通用户 (User)**：SOP 的执行者，负责按照 SOP 规范进行操作，上传视频证明，并获取系统的智能评估与改进建议。

## 3. 核心业务流程
- **SOP 制定流程（管理员）**：
  登录系统 -> 进入管理员工作台 -> 新建 SOP -> 设定该 SOP 的总步数 -> 为每个步骤添加详细的动作描述与规范要求 -> 发布 SOP。
- **SOP 执行与评估流程（用户）**：
  登录系统 -> 进入用户任务台 -> 浏览并选择需要执行的 SOP -> 按照步骤指引，逐一上传对应的操作视频 -> 提交评估 -> 系统后台处理（视频解析、关键帧提取、大模型识别） -> 查看判断结果与改进反馈。

## 4. 页面与功能模块需求

### 4.1 登录与权限模块
- **身份认证**：提供统一的登录页面，支持账号密码登录。
- **角色路由**：登录成功后，系统根据账号角色自动跳转至对应的控制台（管理员端或用户端）。

### 4.2 管理员端 (Admin Dashboard)
- **SOP 列表管理**：
  - 展示当前系统内已创建的所有 SOP 列表。
  - 支持对现有 SOP 的查看、编辑和删除。
- **新建/编辑 SOP**：
  - **基础信息**：输入 SOP 名称、适用场景等基本概述。
  - **步骤设定**：动态设定该 SOP 包含的具体步数（如：3步、5步）。
  - **步骤详情描述**：为每一个步骤配置“标准行为规范输入”。这是后续大模型进行比对的核心依据，需支持输入详细的文字描述（如：“第一步：佩戴护目镜和无菌手套”）。

### 4.3 用户端 (User Dashboard)
- **SOP 任务中心**：
  - **选择 SOP**：展示所有发布的 SOP，用户可浏览并选择当前需要执行的 SOP 任务。
- **视频上传与执行向导**：
  - **步骤指引**：清晰展示当前 SOP 的所有步骤及管理员设定的“步骤描述”，指导用户规范操作。
  - **视频上传**：为每一个步骤提供独立的视频上传入口，支持用户分步上传操作视频片段。
- **评估与反馈报告**：
  - **状态追踪**：视频提交后，展示“视频解析中 -> 关键帧提取中 -> 大模型识别中 -> 评估完成”的进度状态。
  - **结果判定展示**：根据识别结果，展示该次 SOP 执行中各个步骤的顺序和动作是否正确（通过/不通过）。
  - **反馈与建议**：展示大模型生成的详细反馈机制，明确指出哪些动作不符合标准，并提出具体的改进建议，以优化用户未来的行为表现。

## 5. 核心 AI 处理逻辑 (后台/算法端)
此部分为系统的核心技术难点，系统需具备以下自动化处理流水线：
1. **视频解析与分割**：对用户上传的视频进行预处理，提取关键帧，并将长视频精准分割为对应具体动作的短视频片段。
2. **行为识别与比对**：调用视觉大模型（如 GPT-4V, Qwen-VL 等）或专门的行为识别模型，将“分割后的视频片段/关键帧”与管理员输入的“标准行为规范描述”进行多模态理解与比对。
3. **结果判定与生成建议**：大模型基于比对结果，判断每个步骤的顺序和动作是否正确，并利用自然语言生成针对性的改进建议文本。

## 6. 非功能性需求
- **系统性能**：由于涉及视频上传和大模型推理，前端需具备良好的加载动画和异步任务状态轮询机制，避免用户长时间等待造成的体验不佳。
- **易用性 (UI/UX)**：用户端上传流程应采用“向导式（Step-by-Step）”设计，降低用户的理解成本。
- **准确性保障**：需设计良好的 Prompt（提示词）工程，确保大模型能准确理解管理员的 SOP 描述，并结合关键帧给出客观的判定结果。
