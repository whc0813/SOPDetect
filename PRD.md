# 产品需求文档 (PRD)：基于视觉分析的标准操作流程管理系统

## 1. 项目概述
- **项目名称**：基于视觉分析的标准操作流程 (SOP) 管理系统
- **项目类型**：毕业设计
- **项目愿景**：开发一个基于大视觉/语言模型（大模型）的标准行为识别系统。通过智能化的视觉分析技术，实现对人员操作行为的高效、准确识别与正误判断，从而推动相关领域（如工业制造、医疗操作、实验室规范等）的标准操作流程规范化与自动化管理。

## 2. 目标用户与角色
系统主要面向两类用户角色，具有不同的权限和业务流：
- **管理员 (Admin)**：负责制定和维护标准操作流程（SOP），定义每一步骤的具体规范文字描述，并上传标准动作的示范视频。
- **普通用户 (User)**：SOP 的执行者，负责按照 SOP 规范进行操作，录制并上传完整的操作视频，获取大模型智能评估的结果与改进建议。

## 3. 核心业务流程
- **SOP 制定流程（管理员）**：
  登录系统 -> 进入管理员工作台 -> 新建 SOP -> 设定该 SOP 的总步数 -> 为每个步骤添加详细的动作描述**并上传示范视频** -> 发布 SOP。
- **SOP 执行与评估流程（用户）**：
  登录系统 -> 进入用户任务台 -> 浏览并选择需要执行的 SOP -> 按照步骤指引，**上传一段完整的操作视频** -> 提交解析与验证 -> 系统请求多模态大模型（结合管理员示范视频与用户视频进行比对） -> 查看结构化判定结果与详细反馈。

## 4. 页面与功能模块需求

### 4.1 登录与权限模块
- **身份认证**：提供统一的极简登录页面，支持账号密码登录（如 admin/user）。
- **角色路由**：登录成功后，系统根据账号角色自动跳转至对应的控制台（管理员端或用户端）。

### 4.2 管理员端 (Admin Dashboard)
- **SOP 列表管理**：
  - 展示当前系统内已创建的所有 SOP 列表。
  - 支持对现有 SOP 的查看、删除操作。
- **新建 SOP**：
  - **基础信息**：输入 SOP 名称、适用场景等基本概述。
  - **步骤设定**：动态设定该 SOP 包含的具体步数。
  - **步骤详情与示范**：为每一个步骤配置“文字规范描述”，并**必须为每一步上传“标准示范视频”**。这是后续大模型进行比对的核心依据（本地采用 IndexedDB 存储视频）。

### 4.3 用户端 (User Dashboard)
- **SOP 任务与历史中心**：
  - **任务列表**：展示所有发布的 SOP，用户可浏览并选择当前需要执行的 SOP 任务。
  - **执行历史**：记录用户过往的验证历史、最终得分与通过状态。
- **视频上传与执行向导**：
  - **步骤指引**：清晰展示当前 SOP 的所有步骤及管理员设定的“步骤描述”与“示范视频”，指导用户规范操作。
  - **完整视频上传**：用户仅需提供一个**统一的完整操作视频上传入口**，避免分步上传的繁琐，极大提升体验。
- **评估与反馈报告**：
  - **模型配置**：支持前端页面直接配置多模态大模型 API 参数（如阿里云百炼 API Key、Base URL、Model 等），方便演示与调试。
  - **结果判定展示**：大模型返回 JSON 结构化数据后，系统解析并展示最终得分、整体是否通过。
  - **分布明细与建议**：展示每一步骤的判定状态（通过/异常）、得分、置信度，以及大模型生成的具体依据与改进建议。

## 5. 核心 AI 处理逻辑 (后台/算法端)
当前系统的核心逻辑采用**端到端的多模态大模型直接评估**，省去了传统方案中复杂的视频手动分割与关键帧提取流水线：
1. **多模态上下文组装**：提取管理员为 SOP 各步骤上传的“示范视频”和“文本描述”，拼接成带有顺序的标准上下文（Standard Context）。
2. **大模型联合推理**：将用户的“完整长视频”与上述 Standard Context 一同发送给支持视频理解的多模态大模型（如 Qwen-VL 系列等）。
3. **结构化 Prompt 与输出**：通过设计良好的 Prompt（提示词）并强制模型以 JSON Schema 格式输出，让大模型在一次推理中完成所有步骤的对比，直接返回包含：总分、通过状态、总体反馈、各步骤详细评分与证据的结构化 JSON。

## 6. 非功能性需求
- **前端架构与存储**：作为一个纯前端/轻量化毕设项目，SOP 视频资源采用浏览器本地 IndexedDB (sop-demo-db) 进行大文件存储，数据列表使用 LocalStorage，实现无后端轻量化运行。
- **易用性 (UI/UX)**：整体采用现代极简设计风格（类 Apple 风格），具有流畅的过渡动画与卡片式布局，降低用户的理解成本。
- **大模型 API 适配**：系统需具备完善的超时控制、错误捕获与状态提示，以应对大模型视频解析耗时较长的问题，避免用户因长时间等待产生焦虑。
