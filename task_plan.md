# SOP 步骤耗时约束实施计划

## Goal
为 SOP 步骤新增可选最短/最长耗时配置，并通过 `过快完成`、`超时完成` 两类问题参与模型提示、后端规则修正、评分和前端配置。

## Phases
- [x] 后端测试：覆盖新问题类型、罚分、耗时规则和非法时间范围校验
- [x] 后端实现：模型、存储、迁移、Prompt、评分后处理
- [x] 前端实现：管理员步骤表单、详情展示、罚分配置
- [x] 验证：运行后端测试和前端构建检查

## Decisions
- 时间单位统一为秒。
- `null`、空字符串、`0` 均表示未配置该方向的时间限制。
- 不恢复旧 `timingStatus` 字段，仍通过 `issueType` 做统计。
